## What Is a Branch?
A branch is a separate line of development that allows you to work on new features, experiments, or fixes without affecting the main codebase.  
It helps teams collaborate safely and efficiently.

## Why Branching Matters
- Keeps the main branch stable  
- Allows multiple developers to work in parallel  
- Makes it easier to test ideas without breaking anything  
- Supports clean workflows like Git Flow and trunk‑based development

## Commands Used
- `git branch` — list branches  
- `git checkout -b <branch>` — create and switch  
- `git switch <branch>` — switch branches  
- `git merge <branch>` — merge into current branch
